# 개발 단계 로드맵(SSOT)

## Phase 0 — 기준선 고정(지금)
- 서비스 경계/데이터 소유권 확정(user/audition/media/gateway)
- RBAC 모델 문서화 및 최소 구현 범위 확정
- 오디션 심사 단계 “가변” 모델로 정리(하드코딩 제거 계획 수립)
- API 계약(에러/응답/버전) 문서화

## Phase 1 — 오디션 핵심 플로우 완성
- 기획사: 공고 생성/수정/게시
- 지원자: 유튜브 영상으로 지원
- 심사: 단계 이동, 합격/불합격, 히스토리 기록

## Phase 2 — 개인 채널(발견/탐색) 강화
- 지원자 채널(영상/프로필/정렬/공개범위)
- 기획사 탐색 UX(필터/검색/북마크/평가 메모)

## Phase 3 — 트레이너 피드백 + 칼럼(콘텐츠) (완료: 최소 구현)
- 역할: INSTRUCTOR/COLUMNIST 도입
- 피드백 플로우(요청→수락→제공→검수/신고)
- 칼럼 작성/발행/노출 정책

## Phase 4 — 글로벌/결제 확장 (완료: 최소 구현)
- 번역 파이프라인(도메인 콘텐츠 번역)
- Stripe 결제/정산/환불/웹훅 운영
- 국가/통화/세금 이슈 반영

## Phase 5 — 탐색/검색 + 기획사 운영 (완료: 최소 구현)
- 공개 채널 탐색/검색
- 기획사 북마크/평가 메모
- 기본 추천/정렬 기준

## Phase 6 — 운영/안정성 + 성장 지표 (완료: 최소 구현)
- 운영 대시보드/통계
- 감사 로그/이벤트 추적
- 알림/모니터링

## Phase 7 — 운영 알림 + 감사 로그 자동화 (완료: 최소 구현)
- 관리자 알림 저장/조회/확인
- 관리자 요청 자동 감사 로그 기록
