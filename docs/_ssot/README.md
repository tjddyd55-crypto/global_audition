# SSOT Index

이 디렉토리는 Global Audition Platform의 **단일 진실 원천(SSOT)** 문서 모음이다.

## 문서 목록
- `00_PRODUCT_VISION.md`: 제품 비전/핵심 사용자 흐름
- `01_RBAC_ROLES.md`: 역할/권한 체계(RBAC) 초안
- `02_DOMAIN_MODEL.md`: 서비스 경계/도메인 모델/확장 포인트
- `03_I18N_TRANSLATION.md`: i18n/도메인 번역 전략 및 Provider 추상화
- `04_PAYMENTS_STRIPE.md`: Stripe 결제 아키텍처(웹훅 idempotency 등)
- `05_ROADMAP.md`: 개발 단계 로드맵
- `06_DEVELOPMENT_PLAN.md`: 개발 단계 계획서(실행용)
- `07_PHASE1_WORKBREAKDOWN.md`: Phase 1 상세 실행 계획
- `08_PHASE2_WORKBREAKDOWN.md`: Phase 2 상세 실행 계획
- `09_PHASE3_WORKBREAKDOWN.md`: Phase 3 상세 실행 계획
- `10_PHASE4_WORKBREAKDOWN.md`: Phase 4 상세 실행 계획
- `11_PHASE5_WORKBREAKDOWN.md`: Phase 5 상세 실행 계획
- `12_PHASE6_WORKBREAKDOWN.md`: Phase 6 상세 실행 계획
- `13_PHASE7_WORKBREAKDOWN.md`: Phase 7 상세 실행 계획
- `14_ARCHITECTURE_BASELINE.md`: 아키텍처 기준선 (서비스 구조, 하드 규칙)
- `15_DOMAIN_DEFINITION.md`: 도메인 정의 (소유권, 책임)
- `16_API_RULES.md`: API 설계 규칙 (내부/외부 API 분리, Entity 노출 금지)
- `17_FRONTEND_RULES.md`: 프론트엔드 개발 규칙 (기술 스택, 상태 관리)
- `18_FIGMA_RULES.md`: Figma UX 설계 규칙
- `19_DEV_FLOW_RULES.md`: 개발 플로우 규칙 (작업 원칙, 프로세스)

## 운영 원칙
- “기준이 바뀌면” 코드보다 **SSOT 문서가 먼저** 바뀐다.
- 각 기능 추가/변경 PR(또는 커밋)에는 관련 SSOT 업데이트가 포함되어야 한다.
