# Figma 규칙 (SSOT)

이 문서는 Figma를 통한 UX 설계 규칙을 정의한다.

## Figma의 역할
Figma는 **UX 상태 정의 도구**다.
- 디자인 시스템 정의
- 사용자 플로우 시각화
- 컴포넌트 상태 정의

## 필수 페이지 (우선순위)

### 1. Agency Dashboard
- 기획사 대시보드
- 통계 카드 (오디션 수, 지원자 수)
- 최근 오디션 목록
- 빠른 액션 버튼

### 2. Audition Detail
- 오디션 상세 페이지
- 오디션 정보 표시
- 지원자 목록/관리
- 심사 단계별 필터링

### 3. Applicant Management
- 지원자 관리 페이지
- 단계별 필터링
- 일괄 처리 기능
- 개별 합격/불합격 처리

### 4. Profile Edit
- 프로필 수정 페이지
- 기획사 프로필 편집
- 지원자 프로필 편집
- 이미지 업로드

## 디자인 원칙
- 모바일 우선 디자인
- 접근성 고려
- 일관된 디자인 시스템

## 상태 정의
- 각 페이지의 모든 상태 정의
- 로딩 상태
- 에러 상태
- 빈 상태 (Empty State)

## 컴포넌트 정의
- 재사용 가능한 컴포넌트 정의
- 상태별 변형 정의
- 반응형 브레이크포인트 정의
