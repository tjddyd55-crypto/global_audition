# Global Audition — Creative Registry Extension (SSOT)

## 0) 목적
글로벌 오디션 플랫폼에 “창작물(가사/작곡/데모/보컬가이드/AI생성물)”을 안전하게 등록하고,
‘저작권 보호’가 아니라 **“창작물 존재/시점/작성자 주장 근거를 고정(확정)”**하는 기능을 추가한다.

- 저작권 등록 기관 역할 ❌
- 독창성 판단/법적 판단 ❌
- 플랫폼이 제공하는 것: 존재 확인 기록(해시+타임스탬프+계정) + 증거 패키지 출력 ✅

## 1) 핵심 개념
### 1.1 Creative Asset
오디션과 독립적으로 존재하는 자산 단위.
오디션은 파일을 직접 업로드하지 않고 **Asset 참조**만 한다.

AssetType 예시:
- LYRIC (가사 텍스트)
- COMPOSITION (악보/미디/프로젝트 파일)
- DEMO_AUDIO (데모 음원)
- VOCAL_GUIDE (가이드 보컬)
- STEMS (스텝/트랙)
- AI_GENERATED / AI_ASSISTED (선언값)

### 1.2 Creative Registry
Asset 등록 시 생성되는 “존재 확인 기록”.
- content_hash (SHA-256)
- created_at (등록 시각)
- escrow_account_id (또는 creator_identity)
- declared_creation_type (HUMAN / AI_ASSISTED / AI_GENERATED)
- append-only (버전은 새 레코드로)

## 2) 사용자 기능(오디션 플랫폼)
### 2.1 Creative Vault (창작물 보관소)
- 신규 메뉴: “창작물 / Vault”
- 업로드/등록:
  - 파일(오디오/미디/프로젝트 등) 또는 텍스트(가사) 입력
  - 해시 생성(파일/텍스트)
  - 선언값 입력(창작 방식, 권리 선언(선택), 공개 범위)
  - Registry에 기록 후 asset_id 발급
- 공개 범위:
  - PUBLIC: 기획사/검증된 평가자 열람 가능
  - AUDITION_ONLY: 오디션에 첨부한 대상만 열람 가능(기획사/심사자)
  - PRIVATE: 본인만 열람

### 2.2 오디션 지원에 Asset 첨부 (핵심 연결)
- 기존: 파일 업로드 방식 ❌
- 변경: “내 Vault에서 선택” → **asset_id 참조** ⭕
- 지원서에는 attachments: [asset_id...] 형태로만 저장(원본 파일 중복 저장 금지)

### 2.3 평가(전문가 중심)
- 평가 주체: 기획사 계정/인증된 평가자만 가능(일반 유저 제한)
- 평가 항목은 의견/피드백 중심 (권리 판단 금지)
- 평가 공개 범위:
  - 공개(창작자 프로필에 노출)
  - 비공개(창작자만 확인)

## 3) 플랫폼 책임 범위(문구/정책)
### 3.1 금지 문구
- “저작권 보호 보장”
- “법적 효력 보장”
- “원작 판별/독창성 판단”

### 3.2 권장 문구(UX/약관)
- “업로드 즉시 존재 확인 기록이 생성됩니다.”
- “본 플랫폼은 저작권 등록기관이 아니며, 업로더 선언과 기록을 저장합니다.”
- “해당 기록은 분쟁 시 증거 자료로 활용될 수 있습니다.”

## 4) 데이터/연동 요건(오디션 관점)
- 오디션은 asset_id를 참조
- 자산 메타데이터/프리뷰만 표시
- Evidence Export는 “보기/다운로드 링크”만 제공(실제 생성은 에스크로/레지스트리 서비스가 담당)

## 5) MVP 범위
MVP는 “등록 + 오디션 첨부 + 전문가 평가 + 증거 보기”까지.
거래/라이선스/결제는 다음 단계로 분리.

## 6) 확장(차후)
- 콜라보 요청(자산 기반)
- 라이선스 제안/에스크로 결제 연계
- 외부 API 연동(타 오디션/타 서비스)
